# バフ/デバフ辞書パイプライン 実装メモ

## 実装した流れ

1. データ辞書
   - `scripts/build_buff_debuff_pipeline.py` で作成。
   - 競艇場 × 号艇 × データ項目 × 数値帯ごとに、1着率と3着内率を同じ競艇場・同じ号艇の基準値と比較。
   - `バフ / 超バフ / デバフ / 超デバフ / 中立 / サンプル不足` を付与。

2. 朝ランキング
   - 展示前に取れる `AI3連対率`、`一般3連対率`、`AI+順位`、`平均ST順位`、`AI予測%`、`オッズ評価%`、1号艇決まり手率、風波などでスコア化。
   - 出力: `data/output/buff_debuff_ranking_morning_YYYYMMDD.json`

3. 展示後補正
   - 展示タイム、1周タイム、展示+1周平均との差、直線、半周、まわり足、展示ST、チルト、スーパースリットアラートを追加。
   - 出力: `data/output/buff_debuff_ranking_preview_YYYYMMDD.json`

4. 買い目ロジック
   - 頭候補: バフ辞書上の1着上振れが強い3〜6号艇から2艇。
   - 軸候補: AI3連対率+一般3連対率の2位&3位を基本軸。
   - 消し候補: 頭・軸以外で3着内スコアが低い艇。
   - 買い目: 10〜15点、5/6号艇絡みと軸のどちらかを含む三連単。

5. 検証
   - 学習: 2022-04-01〜2025-12-31
   - 検証: 2026-01-01〜2026-06-18
   - 回収率は1点100円均等。的中した全払戻を含む。

## 現時点の判断

- 1号艇飛びと5/6号艇絡みはかなり拾えている。
- ただし万舟率と回収率は現行厳選ランキングを置き換える水準ではない。
- 展示後補正は回収率を上げたが、万舟率は朝版より下がった。
- 本番ランキングへはまだ入れず、既存ランキングの補助ファクターとして使うのが安全。

## 次に改善する点

- 1号艇が飛んでも安いレースを除外するため、人気1号艇/オッズ評価の扱いを強める。
- 5/6絡み率は高いので、5/6が絡んでも配当が跳ねる条件に絞る。
- 展示後補正は、全レースを上げ下げするより「買い/見送り」のフィルターとして使う。
- 頭候補と軸候補が重なる場合の買い目生成を整理する。
- 現行厳選ランキングTOP5に対して、バフ辞書を上乗せした場合の差分検証を追加する。
